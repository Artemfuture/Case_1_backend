# Кейс 1 - Банк, льготные кредиты

REST API сервис для проверки пакетов документов по программам `federal` и `regional`.

## Используемые технологии

- Python 3.11 — основной язык разработки.
- FastAPI — создание REST API с автоматической OpenAPI-документацией и валидацией.
- PostgreSQL — хранение информации о проверках документов.
- SQLAlchemy 2.0 — ORM для работы с базой данных.
- Alembic — управление миграциями базы данных.
- Pydantic — валидация входных и выходных данных.
- pytest — тестирование бизнес-логики.
- Docker / Docker Compose — контейнеризация приложения и базы данных.

## Архитектура

- `app/api` — HTTP-роуты.
- `app/services` — бизнес-логика проверки, определения типов документов и статуса.
- `app/models` — SQLAlchemy-модели.
- `app/schemas` — Pydantic-схемы и enum-типы.
- `migrations` — миграции Alembic.

## Переменные окружения

Пример находится в `.env.example`.

- `DATABASE_URL` — строка подключения к PostgreSQL.
- `DATABASE_ECHO` — включение SQL-лога SQLAlchemy.
- `APP_NAME` — название приложения.
- `APP_VERSION` — версия API.
- `DEBUG` — режим отладки.
- `MAX_FILE_SIZE_MB` — максимальный размер файла.
- `UPLOADS_DIR` — каталог для сохранения файлов.
- `SERVER_HOST` — host для запуска сервера.
- `SERVER_PORT` — port для запуска сервера.

## Запуск

### Через Docker Compose

```bash
docker compose up --build
```

После запуска API доступен на `http://localhost:8000`.

### Локально

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Скопируйте `.env.example` в `.env` и при необходимости измените значения.

3. Примените миграции:

```bash
alembic upgrade head
```

4. Запустите приложение:

```bash
uvicorn app.main:app --reload
```

## Тесты

```bash
pytest
```

## API

- `POST /api/checks` — загрузка файлов и запуск проверки.
- `GET /api/checks` — список проверок.
- `GET /api/checks/{id}` — детальная информация по проверке.
- `GET /health` — проверка доступности сервиса.
